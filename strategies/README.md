# Strategies

`strategies/` 是 Kaka_Quant 中策略研究与执行这条主线的核心目录。

这个目录的定位不是简单放几个策略脚本，而是承载一套更完整的策略工作流：从策略想法整理、样本筛选、轻量回测、复盘迭代，到成熟策略的日常信号输出与飞书推送。

## 这一部分解决什么问题

在真实交易环境里，策略执行往往最怕两件事：

- 交易员在临盘状态下受情绪影响，执行变形
- 策略研究和日常执行脱节，规则没有真正沉淀成系统

这个目录存在的意义，就是把策略从“经验和感觉”逐步转成“规则、记录、回测、推送、执行”的可复用流程。

## 目录职责

这里未来主要承载四类内容：

### 1. 策略研究

- 策略逻辑说明
- 策略触发条件
- 历史样本筛选
- 复盘记录与研究结论

### 2. 轻量回测

- 针对开发中的策略做夜间回测
- 对样本表现进行统计和对比
- 记录参数调整结果
- 形成可回看的研究输出

这里的设计思路是先保持轻量，不急着引入复杂回测框架，而是优先服务真实研究迭代效率。

### 3. 日常执行

- 将成熟策略纳入日常运行清单
- 固定输出信号结果
- 通过配置决定是否推送到指定飞书机器人
- 保证执行链路更稳定、更少依赖人工临场判断

### 4. 策略注册与管理

当前由 [strategy_registry.yaml](d:/code/Kaka_Quant_Portfolio/strategies/strategy_registry.yaml) 作为策略注册表入口，用来管理策略是否启用、如何归类以及后续如何接入执行链路。

## 对整个项目的意义

如果说 `market/` 更偏向指标研究和行情理解，那么 `strategies/` 更偏向把交易方法系统化落地。

这也是 Kaka_Quant 区别于普通数据看板项目的地方：

- 它不仅帮助交易员理解市场
- 也帮助团队把策略研究、回测复盘和信号执行纳入同一套系统

最终目标不是“展示策略页面”，而是逐步形成一套能减少主观误判、降低情绪干扰、提高执行一致性的策略工作流。
